Компиляция в CSS

Для работы с препроцессорами необходимо настроить среду разработки.
Gulp и NodeJS связка:
В папке проекта открываем командную строку (shift + правая кнопка маши -> командная строка)
Сначала в ней инициализируем рабочую среду командой npm init
В процессе выполнения команды дается возможность настроить параметры проекта (можно оставить дефолтные)
В результате выполнения команды появится файл package.json в котором будут прописаны версии пакетов, используемых в проекте.

Далее необходимо установить gulp и gulp-sass в эту же папку командой npm i --save-dev gulp gulp-sass
Команда --save-dev сохраняет версии установленных пакетов в package.json

Затем в той же папке создаем файл "gulpfile.js" и запишем в нем:
var gulp = require('gulp');//подключаем gulp
var sass = require('gulp-sass');//подключаем sass пакет
gulp.task('sass', function(){//создаем таск "sass"
return gulp.src(['sass/**/*.sass', 'sass/**/*.scss'])//берем источник
.pipe(sass({outputStyle: 'expanded'}).on('error', sass.logError))//преобразуем sass в CSS
.pipe(gulp.dest('css'))//выгружаем результат в папку
});
gulp.task('watch', function(){
gulp.watch(['sass/**/*.sass', 'sass/**/*.scss'], ['sass']);//наблюдение за sass файлами в папке sass
});
gulp.task('default', ['watch']);

outputStyle: - свойство задает то как будет выглядеть css файл
'expanded' - развернутый CSS
'nested' - вложенный (по умолчанию)
'compact' - упакованный, компактный вид (селектор и его свойства выводятся в одну строку)
'compressed' - сжатый вид CSS

Далее с этой же папке необходимо создать подпапку "css" и "sass".
В папке "sass" будут находиться исходники, которые затем компилируются в CSS.
В папке "css" создаем файл main.css и аналогично файл main.sass в папке "sass"
Для того, чтобы код из sass файла перенести с CSS, необходимо в открыть командную строку в папке проекта и ввести простую команду gulp

Синтаксис

Есть два варианта написания - синтаксис sass и синтаксис scss.
Синтаксис отступов (sass):
$font-stack: Helvetica, sans-serif
$primary-color: #333
body
__font: 100% $font-stack
__color: $primary-color
Синтаксис scss пишется как простой CSS (со скобками и ;), но расширен дополнительными возможностями sass (переменные и миксины)
$font-stack: Helvetica, sans-serif;
$primary-color: #333;
body{
__font: 100% $font-stack;
__color: $primary-color;
}
Синтаксис sass подразумевает использование или только табов или только двух пробелов! В случае смешения этих правил компилятор выдаст ошибку!
Также систаксисы неявно различаются написанием некоторых функций, надо быть внимательнее при копировании из интернета.

Подключить файл scss в sass можно с помощью директивы import:
@import "файлSCSS"

Правила вложения

sass позволяет создавать иерархические вложения селекторов:
#main p
__color: #00ff00
__width: 97%
__.redbox
____background-color: #ff0000
____color: #000000
Скомпилиованный в CSS этот код будет разделен на 2 сложных селектора:
#main p{
color: #00ff00;
width: 97%;
}
#main p .redbox{
background-color: #ff0000;
color: #000000;
}

Привязка к родительскому селектору

Можно расширить селектор, не создавая нового правила. Для этого используется знак амперсанда (&)
Внешне это выглядит как использование дочернего селектора (как выше в примере), но знак & расширяет именно родительский селектор:
#main
__color: black
__a
____font-weight: bold
____&:hover
______color: red
В CSS скомпилируется так:
#main{
color: black;
}
#main a{
font-weight: bold;
}
#main a:hover{
color: red;
}
Также с помощью амперсанда & можно создавать новые селекторы, добавляя к предыдущим через тире симболы.
#main
__...
__&-sidebar
__...
Создаст в CSS два селектора:
#main{...}
#main-sidebar{...}

Вложенные свойства

Селекторы с дефисом (типа margin-top), имеют общий "корень" - margin, что позволяет записывать их так:
.selector
__margin:
____top: 10px
____bottom: 15px
____left: 20px
____right: 30px
В CSS скомпилируется как:
.selector{
margin-top: 10px;
margin-bottom: 15px;
margin-left: 20px;
margin-right: 30px;
}

Селекторы - шаблоны

Полезны когда необходимо одинаковые стили писать много раз. Их прописывают в селекторе - шаблоне (начинается с %).
Затем с помощью директивы @extend селектор - шаблон просто вставляют в любой другой селектор, со всеми его стилями:
%item-base
__display: inline-block
__transition: all .5s ease
__max-width: 320px
.item-type-one
__@extend %item-base//добавляем шаблон
__margin-top: 20px
.item-type-two
__@extend %item-base//добавляем шаблон
__margin-top: 10px
В CSS скомпилируется как:
.item-type-one, .item-type-two{
display: inline-block;
transition: all .5s ease;
max-width: 320px;
}
.item-type-one{
margin-top: 20px;
}
.item-type-two{
margin-top: 10px;
}

SASSScript значительно расширяет возможности SASS за счет использования пользовательских переменных, арифметических и прочих функций.
Может быть использован для автоматической генерации новых селекторов и свойств.

Переменные sass

В переменных можно хранить цвета, дефолтные значения и константы. Переменные можно использовать в любом месте sass файла.
Переменная создается так: $название: значение
$accent-color: #ff9910

Операции с числами и строками + интерполяция

sass дает возможность использовать стандартные арифметические операции над числами (+ - * / % < <= > >=).
Также есть возможность соединять строки (конкатенация):
$summ: 10+20/2//переменная хранит значение 20
...margin: $summ + px//скомпилируется в CSS как 20px (20 + px)
Используем переменную как свойство. Это интерполяция (использование переменной в как вставку) пишется #{$переменная}
$one: pad
$two: ding
#{$one} + #{$two}: 10px//получится padding: 10px;

Операции с цветами

Цвета в sass можно складывать, вычитать, делить и умножать. Эти операции выполняются для каждого цвета отдельно - красного (#++----), зеленого (#--++--) и синего (#----++).
$color1: #440201 + #550506//складываются поочередно красный (44+55), зеленый и т.д.
Получится цвет #990709.
При операции с rgba цветами (rgba(180, 0, 0, 0.75)) последнее значение (насыщенности) должно быть одинаковым у обоих цветов.

@import

С помощью директивы @import можно импортировать в свой sass код файлы .sass .scss .css
Директива @import избирательна синтаксически и следующие записи воспримет как простой CSS и не импортирует файлы:
@import "foo.css";//не импортирует тк файлы CSS надо импортировать без типа .css
@import "foo" screen;//не импортирует, тк содержит медиа запрос
@import "http://foo.com/bar";//не импортирует ссылки
@import url(foo);//не импортирует через атрибут url
А следующие файлы импортируютс правильно:
@import "media.sass";
@import "media.scss";
@import "media";//тип не указывается если файл тоже обычный CSS

@at-root

Директива @at-root поднимает содержимые в ней правила в корень. Т.е. дочерний селектор прописанный после этой директивы при компиляции перестанет быть дочерним.
.selector
__margin-top: 10px
__@at-root
____.item
______color: #ccc
__.item
____color: #fff
После компиляции в CSS класс .item (с color:#ccc) станет отдельным селектором:
.selector{
margin-top:10px;
}
.item{
color: #ccc;
}
.selector .item{
color: #fff;
}

Директива @if()

sass поддерживает использование выражений и функций для разлчных условий(if), циклов(for) и т.д.
@if - директива для создания условий выполнения:
$var: 5
.selector
__@if #var > 5
____content: 'больше пяти'
__@else if $var <= 5
____content: 'меньше'
__@else
____content: ''
В CSS для селектора .selector значение content будет "меньше".

Директива @for

@for выводит блок со стилями определенное количество раз.
Для этого задается переменная - счетчик и ее рамки инкрементации:
@for $1 from 1 to 12//вместо to можно through если требуется включительно до 12
__.item-#{$i}//интерполяция #{} для встраивания переменной
____color: red

Директива @each

Используется если надо пройтись по списку значений (а не просто чисел):
@each $name in head, content, footer
__.section-#{$name}//интерполяция в селектор
____background-image: url(img/#{$name}.png)//интерполяция в значение свойства
В CSS скомпилируется три разных селектора с разными свойствами:
.section-head{
background-image: url(img/head.png);//переменная head встроится и в селектор и в свойство благодаря интерполяции
}
.section-content{
background-image: url(img/content.png);//переменная content встроится и в селектор и в свойство благодаря интерполяции
}
.section-footer{
background-image: url(img/footer.png);//переменная footer встроится и в селектор и в свойство благодаря интерполяции
}

Директива @while

Циклично выводит блок стилей, пока выражение является true
$i: 7
@while $i > 0
__.item-#{$i}
____width: 1em * $i
__$i: $i - 3

Это блоки sass кода (как шаблоны), которые могут принимать аргументы.
Это похоже на функцию, которая принимает аргумент и выдает в зависимости от него стили.
Создаем миксин:
@mixin border($color)
__border: $color 1px solid
И используем миксин для селектора p:
p
__@include border(#333)
В CSS скомпилируется стиль border: #333 1px solid; для селектора p.
