Регулярные выражения

С помощью регулярных выражений можно производить проверку пользовательского ввода. К примеру на наличие определенных символов (@, _ ...).
В основном применяются при проверки полей форм (email).
Сначала создается переменная - паттерн с комбинацией символов для проверки:
var adr_pattern = /[0-9a-z_]+@[0-9a-z_]+\.[a-z]{2,5}/i;
В этом примере проверки email:
/ - означает что вводиться может множество символов, не только один
[0-9a-z_] - перечисление возможных символов (числа от 0 до 9 и латинские буквы от а до z, также символ _), которые могут находиться до знака "@"
+@ - после символов в [] должна стоять "собачка" - @
+\. - затем должна стоять одна единственная точка (\ - обозначает единство символа)
{2,5} - количество символов в [] перед! {} должно быть от 2 до 5
/i - означает что регистр текста не важен.

Для проверки регулярных выражений в формах используют функцию "test":
if(adr_pattern.test(email)==false){...}
Как альтернативный способ проверки email можно воспользоваться функцией "split":
if(email.split('@').length-1 == 0){...} - функция split разделяет строку (переменная email) на символы до @ и после @.
Функция length вернет в этом случае 1 и отнимаем от нее 1 и сравниваем с 0 - так можно проверить строку на наличие определенного символа (например @).

Инструкция try...catch
Инструкция try...catch состоит из блока try, который содержит одну или несколько инструкций, и блок catch, которые содержит инструкции, определяющие порядок действий при выбросе исключения в блоке try. Иными словами, если в блоке try будет выброшено исключение, то управление будет передано в блок catch. Если в блоке try не возникнет исключений, то блок catch будет пропущен. Блок finally будет выполнен после окончания работы блоков try и catch, вне зависимости от того, было ли выброшено исключение.
В следующем примере вызывается функция getMonthName, которая возвращает название месяца по его номеру. Если месяца с указанным номером не существует, то функция выбросит исключение "InvalidMonthNo", которое будет перехвачено в блоке catch:
function getMonthName(mo) {
mo = mo - 1;// Adjust month number for array index (1 = Jan, 12 = Dec)
var months = ["Jan","Feb","Mar","Apr","May","Jun","Jul", "Aug","Sep","Oct","Nov","Dec"];
if (months[mo]) {
return months[mo];
} else {
throw "InvalidMonthNo";}}//throw keyword is used here
try {// statements to try
monthName = getMonthName(myMonth);}// function could throw exception
atch (e) {
monthName = "unknown";
logMyErrors(e);}// pass exception object to error handler -> your own
Используйте блок catch, чтобы обработать исключения, сгенерированные в блоке try.
catch (catchID) { statements }
JavaScript создает идентификатор catchID, которому присваивается перехваченное исключение, при входе в блок catch; данный идентификатор доступен только в пределах блока catch и уничтожается при выходе из него.
try {
throw "myException"
} catch (e) {
console.error(e);}

Блок finally
Блок finally содержит код, который будет выполнен после окончания работы блоков try и catch, но до того, как будет выполнен код, который следует за инструкцией try...catch. Блок finally выполняется вне зависимости от того, было ли выброшено исключение. Блок finally выполняется даже в том случае, если исключение не перехватывается в блоке catch.
В следующем примере открывается файл, затем в блоке try происходит вызов функции writeMyFile, который может выбросить исключение. Если возникает исключение, то оно обрабатывается в блоке catch. В любом случае файл будет закрыт функцией closeMyFile, вызов которой находится в блоке finally.
openMyFile();
try {
writeMyFile(theData);
} catch(e) {
handleError(e);
} finally {
closeMyFile();}
Если блок finally возвращает значение, то данное значение становится возвращаемым значением всей связки try-catch-finally. Значения, возвращаемые блоками try и catch, будут проигнорированы.

Объект Promise

Начиная с ECMAScript2015, JavaScript поддерживает объект Promise, который используется для отложенных и асинхронных операций.
Объект Promise может находиться в следующих состояниях:

  • ожидание (pending): начальное состояние, не выполнено и не отклонено.
  • выполнено (fulfilled): операция завершена успешно.
  • отклонено (rejected): операция завершена с ошибкой.
  • заданный (settled): обещание выполнено или отклонено, но не находится в состоянии ожидания.

Объекты назначаются переменной как и любые другие типы.

Данные в объекте - это свойства и записываются они через запятую.
Для доступа к свойствам объекта необходимо использовать точку:
spaceship.color;//вернет 'silver'
var paint = spaceship.color;

Второй способ получить значение свойства объекта - через []
Этот способ необходимо использовать тогда, когда свойство объекта записано с пробелом (нельзя spaceship.Active Mission, но можно spaceship['Active Mission'])
Удобно использовать в функциях:
let returnAnyProp = (objectName, propName) =>
objectName[propName];
returnAnyProp(spaceship, color);//вернет 'silver'

После создания объекта можно добавлять ему новые свойства или задавать новые значения сущестующим свойствам:
Но если объект создан как константа (const objname = {type: 'one'}), то мутировать его нельзя (objname = {type:two}), но добавить новые свойства и переназначить их можно (через точку).
spaceship['Fuel Type'] = 'vegetable oil';
spaceship.color = 'gold';
Также можно удалять свойства из объектов используя оператор delete:
delete spaceship.mission;

Метод - это данные в объекте, которые являются функцией.
Свойства - то что содержится в объекте (как переменные в классе), а методы это то что объект может делать (как и методы в классе)
Метод в ES5:
const alienShip = {
invade: function () {
console.log('Hello!')}
};
Метод в ES6:
const alienShip = {
invade () {
console.log('Hello!')}
};
Вызов метода производится как и свойства - через точку:
alienShip.invade();
Методы как и свойства в объекте должны записываться через запятую.

Объекты можно вкладывать друг в друга, создавая сложные конструкции
const spaceship = {
telescope: {
model: '91031-XLT',
focalLength: 2032
},
nanoelectronics: {
computer: {
terabytes: 100,
monitors: 'HD'
},
'back-up': {
battery: 'Lithium',
terabytes: 50
}
}
};
Обращение к свойствам сложных объектов производится так:
spaceship.nanoelectronics['back-up'].battery;

Объекты передаются по ссылке. Это означает, что когда мы передаем переменную, назначенную объекту, в функцию в качестве аргумента, компьютер интерпретирует имя параметра как указывающее на пространство в памяти, содержащее этот объект.
let paintIt = obj => {
obj.color = 'glorious gold'
};
paintIt(spaceship);

Меняем свойства СSS стилей

Взять (get) стили элемента:
var el = document.querySelector('div');
var style = window.getComputedStyle ? getComputedStyle(el, null) : el.currentStyle;// getComputedStyle for modern browsers, currentStyle for IE
console.log(style.backgroundColor);// reading properties
console.log(style['-webkit-text-size-adjust']);/ reading properties containing hyphens

Изменить (set) стили элемента можно ч помощью свойства style объекта:
var el = document.querySelector('div');
el.style.backgroundColor = 'green';
el.style.display = 'none';
el.style['border-radius'] = '5px';
Можно добавить сразу несколько стилей за один раз с помощью cssText:
el.style.cssText += 'background: green; display: none;';

Функция для легкого добавления стилей. Эта вспомогательная функция соответствует методу jQuery $ .css (obj).
function css(el, styles) {
for (var property in styles)
el.style[property] = styles[property];
}
Пример ее использования:
var el = document.querySelector('div');
css(el, { background: 'green', display: 'none', 'border-radius': '5px' });

Ajax

Ajax — это технология, позволяющая странице взаимодействовать с сервером при помощи JavaScript.
Например, когда мы отправляем форму на сайте, значения полей собираются и формируется HTTP запрос, который отправляет данные на сервер. Когда вы постите твит в Твиттер, ваш Твиттер-клиент создает HTTP запрос к серверу API Твиттера и обновляет страницу на основании ответа сервера.
Технология Ajax позволяет отправить данные на сервер без перезагрузки страницы.
С помощью Ajax создается связка JS с PHP, и множество функций (комментирование, поставить лайк, добавить в друзья) выполняются без перезагрузки.

Работа с Ajax

Ajax отправляет данные в PHP и далее на сервер, поэтому необходимо иметь локальный сервер или хостинг.
На локальном сервере в папке сайта необходимо создать папку "WWW" (хороший тон) и в ней всю структуру сайта с index.php
Работать Ajax будет именно с этим PHP файлом.

Загрузка данных на страницу ("Полный контроль")

"Полный контроль" - это использование метода .ajax для отправки данных на сервер, параметры описаны ниже.
Для примера в body нашего PHP файла создадим два элемента:
<p id="load" style="cursor: pointer;">Загрузить данные</p>//будет какбы кнопкой для срабатывания Ajax
<div id="information"> </div>//блок куда будут подгружаться данные

Загрузить данные

Создадим в папке проекта файл contentForAjax.php для обработки запросов и затем в скрипте пишем:
$(document).ready(function(){//скрипт будет работать после загрузки страницы
$("#load").bind("click", function(){//берем в переменную наш элемент с id="load", метод ".bind" - это обработчик событий
$.ajax({//вызываем метод ajax у нашей переменной, он выполняет все действия Ajax (отправка данных, получение итд)
url: "contentForAjax.php",//параметры метода ajax (через запятую)
type: "POST",
data: ({name: 'admin', number: 5}),
dataType: "html",
beforeSend: funcBefore,//необходимо будет описать функции funcBefore() и другие в скрипте(есть в js файле)
success: funcSuccess
});
});
});
В файле "contentForAjax.php" напишем обработчик:
<?php
sleep(2);//затормаживаем отправку данных чтобы показалась функция funcBefore()
echo "Данные:строчка - ".$_POST['name'].", цифра - ".$_POST['number'];//то что отобразится в div#information
?>

Основные параметры метода .ajax. Параметры data и beforeSend не обязательно указывать:
url: "page.php", - на какую страницу будут отправлены данные
type: "POST", - то каким образом мы будем отправлять данные (POST или GET)
data: ({имяПеременной1: "значение1", имяПеременной2: 5}), - с помощью этого параметра передаются переменые
dataType: "html", - определяет какого типа данные передаются (текст или html)
beforeSend: имяФункции, - то что срабатывает до того, как придет ответ от страницы об успешной отправке (функция выполняется, пока идет отправка данных, например гифка загрузки)
success: имяФункцииУспешнойОтправки, - вызывается, когда отправка успешно проведена и получен ответ от страницы

Формат JSON

С помощью JSON можно передавать и получать массивы с сервера.
Для примера создадим в html два selectа со списком стран и городов:

С помощью ajax, jQuery и PHP по выбору страны, во втором селекте будут появляться города этой страны.
Метод JSON.parse() кодирует PHP массив в формат доступный JavaScript.
В примере используем другой метод отправки данных на сервер (не "полный контроль" .ajax) это метод get. Пишем скрипт:
$(document).getElementById('ajaxTut2').ready(function(){
$("select[name='country']").bind("change", function(){//функция выполнится при изменения значения поля select[name='country']
$.get("contentForAjax.php", {country: $("select[name='country']").val()}, function(data){
data = JSON.parse(data);//расшифровываем полученный из php файла ответный массив городов
$("select[name='city']").empty();//очищаем селект с городами перед наполнением его массивом новых городов
for(var id in data){
$("select[name='city']").append($("<option value='" + id "'>" + data[id] + "</option>"));
}
});
});
});
В PHP файле добави две строчки для созврата массива городов во второй селект:
if($_GET["country"] == 1) echo json_encode(array("1" => "Вашингтон", "2" => "Сиэтл"));
else if($_GET["country"] == 2) echo json_encode(array("3" => "Париж", "4" => "Марсель"));
